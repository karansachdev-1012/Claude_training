# Onboarding Checklist

Use this checklist to make sure you've covered all the key areas when onboarding a new developer.

## Must Cover
- [ ] What the project does (from README)
- [ ] How files and folders are organized
- [ ] Where the entry point is (or how skills are triggered, if a skills repo)
- [ ] How to run the project locally
- [ ] Current branch and its purpose

## Cover If Asked
- [ ] Full git history and timeline
- [ ] Who contributed and how much
- [ ] Branching strategy and naming conventions
- [ ] How specific files work internally
- [ ] How to add or modify a skill
