# Project Guidelines

## Commit Standards
- Always use conventional commit format: `feat:`, `fix:`, `docs:`, `chore:`, `refactor:`, `test:`
- Never commit files containing secrets (`.env`, credentials, API keys)
- Write commit messages in imperative mood: "Add feature" not "Added feature"

## Code Style
- Keep skill files (SKILL.md) under 500 lines — use progressive disclosure for large skills
- Every skill must have a `name` and `description` in frontmatter
- Skill descriptions must include trigger phrases that match how users naturally ask

## Repository Rules
- Never force push to `main`
- Always create feature branches for new work
- Branch naming: `feat/`, `fix/`, `docs/`, `chore/` prefixes
